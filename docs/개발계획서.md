# 유사호출부호 경고 시스템 개발계획서

> 작성일: 2026-03-16
> 시스템명: 유사호출부호 경고 시스템 (Similar Callsign Warning System)
> 목적: 항공교통관제(ATC) 환경에서 유사한 호출부호를 가진 항공기 쌍을 실시간 감지하고, 관제사의 오류 보고를 지원하는 웹 기반 경고 시스템

---

## 1. 시스템 개요

### 1.1 배경 및 필요성

항공교통관제 업무 중 유사한 호출부호(예: KAL123 / KAL124)를 가진 항공기가 동시에 관제 구역에 진입할 경우, 관제사가 호출부호를 혼동하여 잘못된 지시를 내릴 위험이 있다. 이러한 유사호출부호 상황을 자동으로 감지하고 관제사에게 실시간으로 경고하여 안전사고를 예방하기 위한 시스템이 필요하다.

### 1.2 시스템 구성

| 구분 | 내용 |
|------|------|
| 서버 | Node.js + Express (포트 4000) |
| 데이터베이스 | Oracle 11g (Thick mode) |
| 프론트엔드 | Vanilla JavaScript (프레임워크 미사용) |
| 배포 환경 | 폐쇄망(Air-gapped Network), Windows 단일 서버 |
| 외부 의존성 | express, oracledb (2개만 사용) |

### 1.3 화면 구성

| 화면 | 접속 URL | 대상 사용자 | 주요 목적 |
|------|----------|-------------|-----------|
| 관제사 화면 | `/` | 관제사 | 실시간 유사호출부호 경고 확인 및 오류 보고 |
| 관리자 화면 | `/admin` | 관리자/안전담당자 | 보고서 관리, 통계 분석, 환경설정 |
| 검출이력 화면 | `/history` | 관제사/관리자 | 과거 검출 데이터 조회 및 분석 |
| 초기설정 화면 | `/setup` | 시스템 관리자 | 배포 환경 설정 가이드 |

## 2. 기능 명세

### 2.1 관제사 화면 (실시간 경고)

#### 2.1.1 실시간 유사호출부호 목록

- 현재 활성 상태인 유사호출부호 쌍을 테이블 형태로 표시
- 설정된 주기(기본 10초)마다 자동 갱신
- 네트워크 오류 시 최대 3회 재시도 (지수 백오프)
- 네트워크 연결 상태를 헤더에 실시간 표시

**표시 항목:**

| 항목 | 설명 |
|------|------|
| 유사호출부호 | 편명1 \| 편명2 형태로 표시 (예: KAL123 \| KAL124) |
| 항공사 | 호출부호 접두어 기반 항공사명 자동 표시 |
| 섹터 | 해당 관제 구역명 |
| 유사도 등급 | 매우높음 / 높음 / 보통 (색상 구분) |
| 오류가능성 | 매우높음 / 높음 / 보통 (색상 구분) |
| 권고사항 | 즉시조치 / 주의관찰 / 정상감시 |
| 보고 상태 | 미보고 / 보고완료 |

#### 2.1.2 섹터 현황 카드

- 전체 관제 섹터를 카드 형태로 한눈에 조회
- 각 섹터별 표시 항목:
  - 현재 관제 중인 항공기 수 (ATFM 연동)
  - 유사호출부호 검출 건수
  - 즉시조치 / 주의관찰 건수
- 섹터 카드 클릭 시 해당 섹터만 필터링
- 전체 섹터 보기 / 개별 섹터 보기 전환

#### 2.1.3 위험도 평가 (3단계)

**유사도 등급 (SIMILARITY 기준):**

| 등급 | 조건 | 표시 색상 |
|------|------|-----------|
| 매우높음 | SIMILARITY ≥ 3 | 빨간색 |
| 높음 | SIMILARITY ≥ 1 | 주황색 |
| 보통 | SIMILARITY < 1 | 파란색 |

**오류발생가능성 등급 (SCORE_PEAK 기준, Excel 반출용):**

| 등급 | 조건 |
|------|------|
| 매우높음 | SCORE_PEAK > 60 |
| 높음 | 45 < SCORE_PEAK ≤ 60 |
| 낮음 | 30 < SCORE_PEAK ≤ 45 |
| 매우낮음 | SCORE_PEAK ≤ 30 |

**관제사 권고사항 (SCORE_PEAK 기준):**

| 권고 | 조건 |
|------|------|
| 즉시조치 | SCORE_PEAK ≥ 70 |
| 주의감시 | 40 ≤ SCORE_PEAK < 70 |
| 일반감시 | 0 < SCORE_PEAK < 40 |
| (빈칸) | SCORE_PEAK = 0 |

#### 2.1.4 오류 보고 기능

- 테이블 행 클릭 시 오류 보고 모달 팝업
- 이미 보고된 건은 수정 모드로 열림

**보고서 입력 항목:**

| 항목 | 입력 방식 | 필수 여부 |
|------|-----------|-----------|
| 보고 일시 | 날짜/시간 선택기 (기본: 현재 시각) | 필수 |
| 유사호출부호 | 자동 표시 (읽기 전용) | - |
| 혼동 편명 | FP1 / FP2 / 양쪽 모두 선택 | 필수 |
| 오류 유형 | 드롭다운 (관리자 설정 연동) | 필수 |
| 세부 오류 유형 | 드롭다운 (상위 유형에 따라 필터링) | 필수 |
| 보고자 | 현재 로그인 사용자 목록 또는 직접 입력 | 필수 |
| 비고 | 자유 입력 (최대 200자) | 선택 |

**오류유형 코드 (TYPE, DB 저장값):**

| 코드 | 오류유형 |
|------|----------|
| 1 | 관제사오류 |
| 2 | 조종사오류 |

**세부오류유형 코드 (TYPE_DETAIL, DB 저장값):**

| 상위유형 | 코드 | 세부오류유형 |
|----------|------|-------------|
| 관제사오류(1) | 1 | 호출부호 발신 오류 |
| 관제사오류(1) | 2 | 오인응답 감지 실패 |
| 관제사오류(1) | - | 기타 |
| 조종사오류(2) | a | 동시응답 |
| 조종사오류(2) | b | 무응답 |
| 조종사오류(2) | c | 오인응답 |
| 조종사오류(2) | - | 기타 |

**혼동 편명 (AO, DB 저장값):**

| 코드 | 의미 |
|------|------|
| 1 | FP1 (첫 번째 항공기) |
| 2 | FP2 (두 번째 항공기) |
| 3 | 양쪽 모두 |

### 2.2 관리자 화면 (보고 이력)

#### 2.2.1 검출 목록 조회

- 기간별 조회: 오늘 / 1주 / 1개월 / 3개월 / 직접입력
- 검색 기준: 검출시각 (시작일시 ~ 종료일시)
- 필터 조건: 오류유형, 섹터, 세부오류유형, 보고여부(전체/보고완료/미보고)
- 환경설정의 유사도 등급 설정이 목록에 자동 적용
- 조회 결과를 테이블로 표시 (보고 유무 관계없이 전체 데이터)
- 행 클릭 시 상세 정보 팝업

**검출 목록 표시 항목:**

| 항목 | 설명 |
|------|------|
| 검출시각 | 유사호출부호 검출 시각 (MM-DD HH:MM) |
| 섹터 | 관제 구역명 |
| 유사호출부호 | 편명1 \| 편명2 |
| 유사도 | 매우높음 / 높음 / 보통 (색상 태그) |
| 권고사항 | 즉시조치 / 주의감시 / 일반감시 |
| 보고 | 보고완료 / - |
| 오류유형 | 관제사오류, 조종사오류 등 (미보고 시 -) |
| 세부오류유형 | 동시응답, 무응답 등 (미보고 시 -) |

#### 2.2.2 보고서 삭제

- 체크박스로 여러 건 선택 후 일괄 삭제
- 삭제 전 확인 대화상자 표시
- 삭제된 건수 결과 표시

#### 2.2.3 통계 현황

**요약 카드:**
- 전체 보고 건수
- 오류유형별 건수 (관제사오류, 조종사오류 등)
- 당일 보고 건수

**통계 차트 (4종):**

| 차트 | 유형 | 내용 |
|------|------|------|
| 섹터별 현황 | 가로 막대 | 섹터별 보고 건수 비교 |
| 월별 추이 | 꺾은선 | 최근 12개월 보고 추이 |
| 오류유형 분포 | 세로 막대 | 유형별 보고 비율 |
| 세부유형 분포 | 세로 막대 | 세부유형별 보고 비율 |

#### 2.2.4 시간대별 검출 현황 (사이드바)

- 오늘 0시~23시까지 시간대별 검출 건수 막대 차트
- 위험도별 검출 건수 (매우높음/높음/보통)
- 활성/해제 구분 건수
- 섹터별 검출 건수 요약

#### 2.2.5 Excel 내보내기

- 전체 호출부호 데이터를 Excel 파일로 다운로드
- 종료된 건만 대상 (활성 건 제외)
- 조회 기간 및 섹터 필터 적용 가능

**Excel 출력 컬럼 (30개):**

| # | 컬럼명 | 소스 | 변환 규칙 |
|---|--------|------|-----------|
| 1 | 시작일시(KST) | DETECTED | UTC→KST 변환 |
| 2 | 종료일시(KST) | CLEARED | UTC→KST 변환 |
| 3 | 관할섹터명 | CCP | 섹터코드→섹터명 변환 |
| 4 | 편명1 | FP1_CALLSIGN | 그대로 |
| 5 | 출발공항1 | FP1_DEPT | 그대로 |
| 6 | 도착공항1 | FP1_DEST | 그대로 |
| 7 | 편명2 | FP2_CALLSIGN | 그대로 |
| 8 | 출발공항2 | FP2_DEPT | 그대로 |
| 9 | 도착공항2 | FP2_DEST | 그대로 |
| 10 | 편명1 \| 편명2 | 조합 | FP1 + ' \| ' + FP2 |
| 11 | 항공사구분 | 접두어 추출 | 동일 시 단일, 상이 시 A \| B |
| 12 | 항공사국문 | T_AIRLINE | CALLSIGN 접두어로 AIRLINE_NAME 조회 |

| 15 | 편명번호동일숫자위치 | MATCH_POS | 0→전체, 1→앞뒤, 2→앞, 3→뒤, 4→가운데 |
| 16 | 편명번호동일숫자갯수 | MATCH_LEN | 그대로 |

| 18 | 편명유사도 | SIMILARITY | ≥3→매우높음, ≥1→높음, ≥0.5→낮음, <0.5→매우낮음 |
| 19 | 최대동시관제량 | CTRL_PEAK | 그대로 |
| 20 | 공존시간(분) | 계산 | (종료일시 - 시작일시) 분 단위, 초 올림 |

| 23 | 관제사권고사항 | SCORE_PEAK | ≥70→즉시조치, ≥40→주의감시, >0→일반감시, 0→빈칸 |
| 24 | 보고여부 | MARK | 1→O, 0→빈칸 |
| 25 | 보고일시(KST) | REPORTED | UTC→KST 변환 |
| 26 | 보고자 | REPORTER | 그대로 |
| 27 | 혼돈편명 | AO→호출부호 | 1→FP1호출부호, 2→FP2호출부호, 3→양쪽 |
| 28 | 오류유형 | TYPE | 1→관제사오류, 2→조종사오류 (미보고 시 빈칸) |
| 29 | 세부오류유형 | TYPE_DETAIL | 설정 기반 매핑 (미보고 시 빈칸) |
| 30 | 비고 | REMARK | 그대로 |

### 2.3 검출이력 화면

#### 2.3.1 이력 조회

- 과거 유사호출부호 검출 데이터 전체 조회
- 페이지네이션 지원 (대량 데이터 처리)

**필터 조건:**

| 필터 | 선택 항목 |
|------|-----------|
| 기간 | 오늘 / 1주 / 1개월 / 3개월 / 직접입력 |
| 섹터 | 전체 / 개별 섹터 |
| 위험도 | 전체 / 매우높음 / 높음 / 보통 |
| 상태 | 전체 / 활성 / 해제 |

**목록 표시 항목:**

| 항목 | 설명 |
|------|------|
| 검출시각 | 유사호출부호 최초 감지 시각 |

| 유사호출부호 | 편명1 vs 편명2 |
| 항공사 | 항공사명 |

| 동시관제량 | 해당 시점 섹터 내 항공기 수 |
| 공존시간 | 두 편명의 동시 관제 시간 (분) |
| 경로 | 출발→도착 공항 |
| 권고사항 | 즉시조치 / 주의관찰 / 정상감시 |
| 보고 | 보고 건수 |

#### 2.3.2 통계 요약 카드

- 전체 검출 건수

#### 2.3.3 일별 추이 차트

- 조회 기간 내 일별 검출 건수 꺾은선 차트
- 차트 표시/숨기기 토글

#### 2.3.4 Excel 내보내기

- 현재 조회된 이력 데이터를 Excel 파일로 다운로드

### 2.4 환경설정 기능 (관리자 화면 내)

#### 2.4.1 기본 설정

| 설정 항목 | 설명 | 기본값 |
|-----------|------|--------|
| 표시 섹터 | 관제사 화면에 표시할 섹터 선택 | 전체 |
| 유사도 표시 수준 | 표시할 위험도 등급 선택 | 매우높음 |
| 자동 갱신 주기 | 관제사 화면 폴링 주기 | 10초 |
| 최대 표시 건수 | 한 화면에 표시할 최대 행 수 | 100건 |

#### 2.4.2 위험도 임계값 설정

| 설정 항목 | 설명 | 기본값 |
|-----------|------|--------|
| 유사도 매우높음 기준 | SIMILARITY ≥ 이 값이면 매우높음 | 2 |
| 유사도 높음 기준 | SIMILARITY ≥ 이 값이면 높음 | 1 |
| 오류가능성 매우높음 기준 | SCORE_PEAK ≥ 이 값이면 매우높음 | 40 |
| 오류가능성 높음 기준 | SCORE_PEAK ≥ 이 값이면 높음 | 20 |

#### 2.4.3 섹터 매핑 관리

- 섹터 코드 ↔ 섹터명 매핑 테이블
- 고정 표시 섹터 설정 (항상 화면에 표시할 섹터)
- 행 추가/삭제 기능

#### 2.4.4 오류 유형 관리

- 오류 유형 항목 추가/삭제 (코드, 명칭)
- 세부 오류 유형 항목 추가/삭제 (코드, 명칭, 상위 유형 연결)
- 설정 변경 시 관제사 화면 보고 모달의 드롭다운에 자동 반영

#### 2.4.5 설정 전파

- 관리자가 설정을 저장하면 서버 파일(settings.json)에 기록
- 관제사 화면은 다음 폴링 시 변경된 설정을 자동으로 반영
- 별도의 재시작 없이 실시간 적용

---

### 2.5 보고자 조회 기능

- 현재 관제 시스템에 로그인된 사용자 목록을 자동 조회
- T_ARTCNT_LOG_HISTORY 테이블에서 로그인/로그아웃 기록을 MINUS 연산하여 현재 접속자 추출
- ATFM_LOGIN 테이블에서 사용자 ID → 이름 매핑
- 오류 보고 시 보고자 드롭다운에 자동 표시
- 조회 실패 시 직접 입력 모드로 전환

---

### 2.6 항공사 정보 연동

- T_AIRLINE 테이블에서 호출부호 접두어 → 항공사 국문명 매핑
- 호출부호에서 영문 접두어 자동 추출 (예: KAL831 → KAL → 대한항공)
- 관제사 화면, 검출이력, Excel 내보내기에서 항공사명 표시
- 전체 1,529개 항공사 코드 등록, 국내 주요 항공사 매핑 완료

    ↓

## 4. 데이터베이스 구조

### 4.1 사용 테이블

| 테이블명 | 구분 | 용도 |
|----------|------|------|
| T_SIMILAR_CALLSIGN_PAIR | 자체 | 유사호출부호 쌍 검출 데이터 (메인 데이터) |
| T_SIMILAR_CALLSIGN_PAIR_REPORT | 자체 | 관제사 오류 보고서 |
| T_AIRLINE | 외부 | 항공사 코드 ↔ 국문명 매핑 |
| T_ARTCNT_LOG_HISTORY | 외부 | 관제사 로그인/로그아웃 이력 |
| ATFM_LOGIN | 외부 | 관제사 ID ↔ 이름 매핑 |
| ATFM_FLIGHTPLAN | 외부 | 현재 비행 중인 항공기 정보 (관제량 조회) |

### 4.2 주요 데이터 규칙

- 활성 데이터 식별: CLEARED = '9999-12-31 23:59:59'
- 시간 저장 형식: 'YYYY-MM-DD HH24:MI:SS' (UTC)
- 화면 표시: UTC → KST 자동 변환 (+9시간)
- 보고서 복합키: IDX + REPORTED (동일 건에 대해 여러 보고 가능)

## 5. 보안 및 안정성

### 5.1 보안 대책

| 구분 | 적용 내용 |
|------|-----------|
| SQL Injection 방지 | 모든 쿼리에 바인드 변수(:param) 사용 |
| XSS 방지 | 사용자 입력 HTML 이스케이프 처리 |
| 보안 헤더 | CSP, X-Frame-Options, X-XSS-Protection 설정 |
| 요청 제한 | API 분당 100회, 관리자 API 분당 60회 |
| 입력 검증 | 날짜 형식, 문자열 길이, 숫자 범위 검증 |

### 5.2 안정성 대책

| 구분 | 적용 내용 |
|------|-----------|
| DB 연결 풀 | 최소 2개 ~ 최대 10개 연결 유지 |
| 네트워크 재시도 | 실패 시 최대 3회 재시도 (지수 백오프) |
| 서버 자동 재시작 | Node.js 프로세스 종료 시 5초 후 자동 재시작 |
| 중복 실행 방지 | 포트 점유 확인 및 lock 파일 기반 중복 차단 |
| 정상 종료 | SIGINT/SIGTERM 수신 시 DB 풀 정리 후 종료 |
| 외부 테이블 오류 대응 | 외부 테이블 미존재 시에도 빈 결과 반환 (시스템 중단 없음) |

## 6. 배포 및 운영

### 6.1 배포 환경

- 폐쇄망(Air-gapped Network) 단일 서버 운영
- 인터넷 환경에서 패키지 설치 후 전체 폴더 복사 방식 배포
- Oracle Instant Client 11.2 사전 설치 필요

### 6.2 실행 방법

| 방법 | 설명 |
|------|------|
| 수동 실행 | start.bat 더블클릭 |
| 서비스 등록 | register_service.bat 관리자 권한 실행 (최초 1회) |
| 서비스 동작 | 윈도우 로그인 시 start_minimized.vbs 자동 실행 |

### 6.3 서비스 운영 구조

```
윈도우 로그인
    ↓
스케줄러가 start_minimized.vbs 실행
    ↓
VBS가 start.bat을 최소화 모드로 실행 (중복 실행 방지)
    ↓
start.bat이 node server.js 실행
    ↓
서버 비정상 종료 시 → 5초 후 자동 재시작 (무한 루프)
CMD 창 닫힘 시 → VBS가 감지하여 재실행
```
