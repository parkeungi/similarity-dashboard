## 1. 프로젝트 개요

항공교통관제(ATC) 환경에서 유사한 호출부호를 가진 항공기 쌍을 실시간으로 감지하고, 관제사가 오류 보고서를 제출할 수 있는 웹 기반 모니터링 시스템.

- **기술 스택**: Node.js + Express, Oracle 11g (Thick mode), Vanilla JavaScript
- **배포 환경**: 폐쇄망(air-gapped network), Windows 단일 서버
- **의존성**: express 4.18, oracledb 6.0 (2개만 사용)
- **포트**: 4000

## 4. 서버 아키텍처

### 4.1 server.js (엔트리 포인트)
```
Express 서버 시작 → DB 연결 풀 초기화 → 라우트 등록 → 포트 4000 리슨
```

**주요 미들웨어 체인:**
1. 보안 헤더 (CSP, X-Frame-Options, X-XSS-Protection 등)
2. `express.json({ limit: '50mb' })` - JSON 파서 (CSV 업로드 대비 큰 제한)
3. `express.static('public')` - 정적 파일 서빙
4. `express.static('tools')` - 도구 파일 서빙 (`/tools` 경로)
5. Rate Limiting → 라우터

**Rate Limiting (인메모리 구현):**
- `/api/*`: 분당 100회
- `/api/admin/*`: 분당 60회
- IP별 윈도우 기반, 5분마다 만료 항목 정리
- X-RateLimit-Limit, X-RateLimit-Remaining 헤더 반환

**Graceful Shutdown:**
- SIGINT/SIGTERM 시그널 처리
- Rate Limit 정리 interval 해제 → HTTP 서버 닫기 → DB 풀 종료
- 10초 타임아웃 후 강제 종료

### 4.2 config/database.js (Oracle 연결 관리)
- Oracle 11g **Thick mode** (Instant Client: `C:\instantclient_11_2`)
- 연결 풀: min 2, max 10, 유휴 60초, ping 60초, SQL 캐시 30
- 단일 풀 (별칭 없음, `oracledb.getConnection()`으로 획득)
- `getPoolStatus()`: 모니터링용 풀 상태 조회 (현재 미사용)

### 4.3 config/timeUtil.js (시간 변환)
- DB는 UTC로 저장, 프론트엔드는 KST 표시
- `kstToUtc()`: 필터 날짜 → DB 쿼리용 UTC 변환
- `utcToKst()`: DB 결과 → 프론트엔드 표시용 KST 변환
- `convertRowsToKst()`: 결과 행 배열의 DETECTED/CLEARED 일괄 변환
- 센티넬 값 `'9999-12-31 23:59:59'`는 변환 제외

### 4.4 config/settings.json (런타임 설정)
서버 파일시스템에 JSON으로 저장. 관리자 화면에서 POST /api/config로 수정.
모든 관제사 브라우저가 다음 폴링 시 자동 반영.

```json
{
  "displaySectors": ["2","3","9","10","11","12","13"],
  "displaySimilarity": ["critical"],
  "refreshRate": 10000,
  "maxRows": 100,
  "sectorMap": { "1": "WH", "2": "GH", ... },
  "fixedSectors": ["1","2","3","4","5","7","8","9","10","11","12","13","14"],
  "errorTypes": [{ "value": 1, "label": "관제사 오류" }, ...],
  "errorDetailTypes": [{ "value": 1, "label": "동시응답", "parentType": 1 }, ...],
  "thresholds": {
    "similarity": { "critical": 2, "caution": 1 },
    "scorePeak": { "critical": 40, "caution": 20 }
  }
}
```

## 5. API 엔드포인트 상세

### 5.1 관제사용 API (`routes/api.js` → `/api`)

| Method | Endpoint | 설명 | 주요 로직 |
|--------|----------|------|-----------|
| GET | /api/config | 환경설정 조회 | settings.json 읽기, 유효성 검사 후 반환 |
| POST | /api/config | 환경설정 저장 | 입력값 검증(sectorMap/errorTypes 등) → settings.json 쓰기 |
| GET | /api/callsigns | 유사호출부호 목록 | 활성 데이터(CLEARED='9999...') 조회, 유사도 필터, 섹터 필터, REPORT_COUNT 서브쿼리 |
| GET | /api/sectors | 섹터별 건수 | 활성 데이터의 CCP별 COUNT + 유사도 필터 |
| GET | /api/reporters | 로그인 사용자 목록 | A_REALTIME_LOGIN에서 오늘 06:00~22:00 로그인 사용자 조회 |
| GET | /api/hourly-stats | 시간대별 통계 | KST 기준 오늘 0~23시 시간대별 검출 건수 |
| GET | /api/control-counts | 관제 건수 | ATFM_FLIGHTPLAN에서 현재 관제 중인 항공기 수 (섹터별) |
| GET | /api/reports/:idx | 특정 보고서 조회 | 수정 모드용 최신 보고서 1건 |

| PUT | /api/reports | 보고서 수정 | 입력 검증 → UPDATE (IDX+REPORTED 복합키) |

## 7. 위험도 평가 체계 (3단계)

### 기준값 (settings.json에서 동적 설정 가능)

| 지표 | 매우높음 (danger/critical) | 높음 (warning/caution) | 보통 (info/monitor) |
|------|---------------------------|----------------------|-------------------|
| 유사도 (SIMILARITY) | > 2 (초과) | > 1 (초과) | <= 1 |
| 오류가능성 (SCORE_PEAK) | >= 40 (이상) | >= 20 (이상) | < 20 |

### 권고사항 (종합 판정)
- **즉시조치**: 유사도 매우높음 **또는** 오류가능성 매우높음
- **주의관찰**: 유사도 높음 **또는** 오류가능성 높음 (즉시조치 미해당)
- **정상감시**: 나머지 전부

### UI 표현
- 즉시조치: 빨간 좌측 보더 + 깜빡이는 애니메이션 (`tr.high-risk`)
- 주의관찰: 노란 좌측 보더 (`tr.med-risk`)
- 정상감시: 기본 스타일

## 10. 보안 분석

### 적용된 보안 조치
| 항목 | 구현 |
|------|------|
| XSS 방지 | `escapeHtml()` 함수로 모든 사용자 입력/DB 데이터 이스케이프 |
| SQL Injection 방지 | 모든 쿼리에 bind 변수(`:paramName`) 사용 |
| 보안 헤더 | CSP, X-Frame-Options, X-XSS-Protection, Referrer-Policy |
| Rate Limiting | IP 기반 인메모리 Rate Limiter (API 100회/분, Admin 60회/분) |
| 에러 정보 노출 방지 | `safeErrorMessage()` - 내부 오류는 서버 로그만, 클라이언트에는 일반 메시지 |
| 입력 검증 | 모든 POST/PUT 엔드포인트에서 타입/범위/길이 검증 |
| DB 연결 관리 | try/finally 패턴으로 연결 반드시 반환 |

### 의도적 미적용 (폐쇄망 전제)
| 항목 | 이유 |
|------|------|
| 인증/인가 없음 | 폐쇄망 단일 서버, 물리적 접근 제한 |
| CORS 미설정 | 같은 서버에서 static + API 제공 |
| HTTPS 미적용 | 내부망 전용 |

### 주의 사항
- `config/database.js`에 DB 자격증명이 하드코딩됨 (`cssown/cssadmin`) - 폐쇄망이므로 허용하지만, `database.js.example` 템플릿이 별도로 존재
- `/api/admin/kill-switch`에서 `taskkill /F /IM powershell.exe` 실행 - 모든 PowerShell 프로세스를 종료함 (대상이 넓음)
- `express.json({ limit: '50mb' })` - 불필요하게 큰 제한 (현재 CSV 업로드 기능은 없음)

## 11. 코드 품질 분석

### 잘된 점
1. **일관된 DB 연결 패턴**: 모든 라우트에서 `let conn; try/catch/finally` 패턴 준수
2. **공통 코드 분리**: common.js로 중복 최소화, 섹터 변경 시 한 곳만 수정
3. **동적 설정 시스템**: settings.json 기반으로 섹터맵/오류유형/기준값을 런타임에 변경 가능
4. **방어적 코딩**: 기준값 정규화(`normalizeThresholds`), NaN/undefined 방어, 기본값 fallback
5. **이벤트 위임**: main.js에서 tbody 단일 리스너로 행 클릭 처리 (메모리 효율적)
6. **재시도 로직**: `fetchWithRetry()` - 지수 백오프 3회 재시도
7. **Graceful shutdown**: 시그널 핸들링 + 타임아웃 강제 종료

### 개선 여지
1. **normalizeThresholds 중복**: `routes/api.js`, `routes/history.js`, `public/js/common.js`, `public/js/admin.js` 4곳에 거의 동일한 함수가 존재
2. **safeErrorMessage 중복**: `api.js`, `admin.js`, `history.js` 3곳에 동일 함수
3. **admin.js 크기**: 1287줄로 가장 큼 - 환경설정/보고서/통계/차트/사이드바가 하나의 파일에 혼재
4. **REPORTER 컬럼 크기**: `VARCHAR2(2)`로 정의되어 있으나 실제 한글 이름 저장 (setup_db.js 기준) - Oracle VARCHAR2 바이트/문자 설정에 따라 문제 가능
5. **stats 쿼리 6회 순차 실행**: admin.js의 `/api/admin/stats`에서 6개 쿼리를 순차 실행 - 단일 쿼리로 합칠 여지 있음
6. **유사도 필터 SQL 문자열 연결**: `buildSimilarityFilter()`에서 기준값을 SQL 문자열에 직접 삽입 (bind 변수 미사용) - 값이 normalizeThresholds를 거치므로 숫자만 가능하여 현재는 안전하나, bind 변수 사용이 더 안전

## 12. 도구 파일 (tools/)

### right_click_switch.ps1
- **목적**: 전체화면 앱(관제 시스템) 위에서 마우스 우클릭 시 유사호출부호 브라우저를 앞으로 가져옴

### alt_tab.ps1
- **목적**: 마우스 사이드 버튼(XButton1)을 Alt+Tab으로 매핑
- **구현**: Low-level Mouse Hook으로 XButton1 이벤트 가로채기 → keybd_event로 Alt+Tab 시뮬레이션

### right_click_switch.bat
- PowerShell 스크립트를 숨김 창 모드(`-WindowStyle Hidden`)로 실행하는 래퍼

## 15. 배포 체크리스트

1. Node.js v18+ 설치
2. Oracle Instant Client 11.2 → `C:\instantclient_11_2`
