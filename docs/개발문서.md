> 최종 수정: 2026-03-27
> 시스템명: 유사호출부호 경고 시스템 (Similar Callsign Warning System)

## 1. 시스템 개요

항공교통관제(ATC) 환경에서 유사한 호출부호를 가진 항공기 쌍을 실시간으로 감지하고, 관제사의 오류 보고를 지원하는 웹 기반 경고 시스템.

| 구분 | 내용 |
|------|------|
| 서버 | Node.js + Express (포트 4000) |
| 데이터베이스 | Oracle 11g (Thick mode, `C:\instantclient_11_2`) |
| 프론트엔드 | Vanilla JavaScript (프레임워크 미사용) |
| 배포 환경 | 폐쇄망(Air-gapped Network), Windows 단일 서버 |
| 외부 의존성 | express 4.18, oracledb 6.0 (2개만 사용) |

### 화면 구성

| 화면 | URL | 대상 | 목적 |
|------|-----|------|------|
| 관제사 화면 | `/` | 관제사 | 실시간 유사호출부호 경고 + 오류 보고 |
| 관리자 화면 | `/admin` | 관리자/안전담당자 | 보고서 관리, 통계, 환경설정 |
| 검출이력 화면 | `/history` | 관제사/관리자 | 과거 검출 데이터 조회 |
| 초기설정 화면 | `/setup` | 시스템 관리자 | 배포 환경 설정 가이드 |

## 4. 서버 아키텍처

### server.js

```
Express 서버 시작 → DB 연결 풀 초기화 → 라우트 등록 → 포트 4000 리슨
```

**미들웨어 체인:**
1. 보안 헤더 (CSP, X-Frame-Options, X-XSS-Protection 등)
2. `express.json()` — JSON 파서
3. `express.static('public')` — 정적 파일
4. Rate Limiting → 라우터

**Rate Limiting (IP 기반 인메모리):**
- `/api/*`: 분당 100회
- `/api/admin/*`: 분당 60회

**Graceful Shutdown:**
- SIGINT/SIGTERM → HTTP 서버 닫기 → DB 풀 종료 → 10초 후 강제 종료

### config/database.js

- Oracle 11g Thick mode
- 연결 풀: min 2, max 10, 유휴 60초, ping 60초

모든 라우트 핸들러는 반드시 이 패턴을 따를 것:

```javascript
let conn;
try {
    conn = await db.getConnection();
    // SQL 실행
} catch (err) {
    // 에러 처리
} finally {
    if (conn) await conn.close();  // 반드시 반환
}
```

### config/timeUtil.js

- DB 저장: UTC 문자열 (`'YYYY-MM-DD HH24:MI:SS'`)
- 화면 표시: KST (UTC+9)
- `kstToUtc()` / `utcToKst()` / `convertRowsToKst()` 사용
- 센티넬 값 `'9999-12-31 23:59:59'`는 변환 제외

### config/settings.json

관리자 POST /api/config 로만 변경. 직접 파일 수정 금지.

```json
{
  "displaySectors": ["2","3","9","10","11","12","13"],
  "displaySimilarity": ["critical"],
  "refreshRate": 10000,
  "maxRows": 100,
  "sectorMap": { "1": "WH", "2": "GH", ... },
  "errorTypes": [...],
  "errorDetailTypes": [...],
  "thresholds": {
    "similarity": { "critical": 2, "caution": 1 },
    "scorePeak": { "critical": 40, "caution": 20 }
  }
}
```

### 외부 테이블 (읽기 전용)

| 테이블 | 용도 |
|--------|------|
| A_REALTIME_LOGIN | 로그인 사용자 목록 (보고자 드롭다운) |
| ATFM_FLIGHTPLAN | 현재 관제 중인 항공기 수 (섹터 카드) |

### Oracle 11g 제약사항

- `FETCH FIRST N ROWS ONLY` 사용 불가 → `WHERE ROWNUM <= N` 또는 서브쿼리 래핑
- `LISTAGG ... WITHIN GROUP` 4000바이트 제한
- 날짜는 문자열 저장, `TO_DATE()` 변환 필수

## 9. 항공사 제출 탭 — 호출부호 쌍 정규화

### 배경

DB의 `T_SIMILAR_CALLSIGN_PAIR`에서 같은 항공기 쌍이 검출 타이밍에 따라 FP1/FP2 순서가 다르게 저장될 수 있음. 목록 표시 및 Excel 반출 시 일관된 순서로 표시하기 위해 정규화 처리를 적용함.

### 정렬 규칙

1. 국내 항공사 + 외항사 쌍 → **국내 항공사가 항상 앞(FP1)**
2. 같은 그룹(국내끼리 / 외항사끼리) → **알파벳 오름차순**

### 국내 항공사 코드 (`public/js/admin.js` — admin 전용)

```javascript
const DOMESTIC_PREFIXES = new Set([
    'KAL','AAR','JJA','JNA','TWB','ABL','ASV','APZ','ESR','EOK','AIH','PTA'
]);
```

> 신규 국내 항공사 추가 시 이 Set만 수정.

### 구현 위치 및 함수

**파일**: `public/js/admin.js` (항공사 제출·Excel 반출 전용, 다른 기능에 영향 없음)

```javascript
// 정렬 우선순위 비교
function compareCallsignOrder(a, b) {
    const da = DOMESTIC_PREFIXES.has(extractAirlinePrefix(a)) ? 0 : 1;
    const db = DOMESTIC_PREFIXES.has(extractAirlinePrefix(b)) ? 0 : 1;
    if (da !== db) return da - db;
    return a <= b ? -1 : 1;
}

// 행 정규화: FP1 > FP2 우선순위이면 관련 필드 전체 교환
function normalizeAirlineRow(r) {
    if (compareCallsignOrder(r.FP1_CALLSIGN || '', r.FP2_CALLSIGN || '') <= 0) return r;
    const ao = r.AO;
    return {
        ...r,
        FP1_CALLSIGN: r.FP2_CALLSIGN, FP1_DEPT: r.FP2_DEPT, FP1_DEST: r.FP2_DEST, FP1_AIRLINE: r.FP2_AIRLINE,
        FP2_CALLSIGN: r.FP1_CALLSIGN, FP2_DEPT: r.FP1_DEPT, FP2_DEST: r.FP1_DEST, FP2_AIRLINE: r.FP1_AIRLINE,
        AO: ao === 1 ? 2 : ao === 2 ? 1 : ao
    };
}
```

### 스왑 대상 필드

| 필드 | 처리 |
|------|------|
| FP1/FP2 CALLSIGN | 직접 교환 |
| FP1/FP2 DEPT, DEST | 직접 교환 |
| FP1/FP2 AIRLINE | 직접 교환 → 항공사구분·항공사국문 열 자동 반영 |
| AO (오류항공기) | 1↔2 반전, 3 유지 → 혼돈편명 열 자동 반영 |

### 적용 위치

`loadAirlineData()` 에서 데이터 로드 직후 적용:

```javascript
AIRLINE_DATA_RAW = filterReportsBySimilarity(result.data || []).map(normalizeAirlineRow);
```

DB 데이터는 변경 없음. 화면 렌더링 및 Excel 저장 단계에서만 정규화됨.
